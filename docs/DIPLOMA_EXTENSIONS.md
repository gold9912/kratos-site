# Что дополнить под диплом

## Обязательная сильная часть

- Раздел требований: функциональные и нефункциональные требования.
- ER-диаграмма Supabase/PostgreSQL.
- Диаграмма компонентов.
- Диаграммы последовательности для заявки, регистрации и отзыва.
- Swagger/OpenAPI как документированный контракт.
- Описание RLS и модели безопасности.
- Docker Compose и CI как часть эксплуатационной документации.

## Admin-extension

Для более убедительной дипломной работы стоит добавить закрытую админ-панель:

- список заявок;
- статусы заявок: новая, в работе, выполнена, отменена;
- модерация отзывов: pending, published, rejected;
- редактирование услуг и цен;
- просмотр статистики: количество заявок, средний рейтинг, популярные услуги.

## Аналитика

Можно добавить отдельный модуль dashboard:

- конверсия заявок;
- средний чек по калькулятору;
- распределение услуг по категориям;
- динамика отзывов.

## Тестирование

Для защиты хорошо показать:

- unit tests для калькулятора;
- e2e tests для auth/orders/reviews;
- smoke-тест frontend в браузере;
- CI pipeline на GitHub Actions.
